Enter planning mode. Before writing any code:

1. **Explore**: Thoroughly read the relevant files to understand existing patterns and architecture
2. **Identify**: List all files that will need to change and why
3. **Consider**: Evaluate multiple approaches with their trade-offs
4. **Design**: Create a concrete, step-by-step implementation plan with:
   - Exact files to create or modify
   - Order of changes (respecting dependencies)
   - Key design decisions and rationale
5. **Present**: Show the plan to the user and wait for explicit approval before proceeding

Do NOT write or edit any files until the user approves the plan. This is a read-only exploration and design phase.
